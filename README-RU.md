# universo-platformo-vue

Реализация Universo Platformo / Universo MMOOMM / Universo Kiberplano на базе Vue / Django и связанного стека на TypeScript / Python

## Структура документации

Проект следует стандартизированной структуре документации:

### Директория `.specify/`

Вся проектная документация и спецификации организованы в директории `.specify/`:

- **`.specify/memory/`** - Содержит конституцию проекта (`constitution.md`), определяющую основные принципы, технологический стек и рабочий процесс разработки
- **`.specify/specs/`** - Содержит спецификации функций, организованные по номеру функции (например, `001-project-initialization/`)
  - Каждая директория функции включает:
    - `spec.md` - Спецификация функции с требованиями и пользовательскими историями
    - `tasks.md` - Задачи реализации, организованные по фазам
    - `plan.md` - Технический план реализации
    - Дополнительные проектные документы по необходимости
- **`.specify/scripts/`** - Скрипты автоматизации для рабочего процесса разработки на основе спецификаций
  - `bash/` - Shell-скрипты для управления функциями и валидации
- **`.specify/templates/`** - Шаблоны для артефактов документации
  - `spec-template.md` - Шаблон спецификации функции
  - `tasks-template.md` - Шаблон списка задач
  - `plan-template.md` - Шаблон плана реализации
  - `checklist-template.md` - Шаблон чеклиста
  - `agent-file-template.md` - Шаблон конфигурации агента

### Конфигурация агентов

Конфигурации агентов хранятся в `.github/agents/` и ссылаются на структуру `.specify/` для всех операций, связанных с функциями.
